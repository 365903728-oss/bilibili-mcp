# Contributing to Bilibili MCP

感谢你对 Bilibili MCP 项目的关注！欢迎贡献代码、报告问题或提出建议。

## How to Contribute

### 报告问题 (Bug Reports)

如果你发现了 bug，请：

1. 检查 [Issues](https://github.com/365903728-oss/bilibili-mcp/issues) 确认问题尚未被报告
2. 创建新的 Issue，包含：
   - 清晰的标题
   - 详细的问题描述
   - 复现步骤
   - 预期行为 vs 实际行为
   - 环境信息（Node 版本、操作系统等）

### 功能建议 (Feature Requests)

1. 检查现有 Issues 确认建议未提出
2. 创建新的 Issue，描述你希望添加的功能
3. 说明该功能的用途和预期效果

### 提交代码 (Pull Requests)

#### 开发流程

1. Fork 本仓库
2. 创建你的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启 Pull Request

#### 代码规范

- 使用 TypeScript
- 遵循现有代码风格
- 添加必要的注释
- 确保代码通过编译 (`npm run build`)
- 测试新功能

#### Commit 信息

使用清晰的 commit 信息：
```
feat: 添加新功能
fix: 修复 bug
docs: 更新文档
style: 代码格式调整
refactor: 重构代码
test: 添加测试
chore: 其他杂项
```

## Legal/Compliance Guidelines

### 重要提醒

在贡献代码时，请确保：

1. **遵守 Bilibili 服务条款** - 使用公开 API，不违反使用规范
2. **尊重版权** - 不添加任何可能侵犯版权的功能
3. **保护隐私** - 不收集、存储用户数据
4. **控制请求频率** - 避免高频请求导致 API 限制
5. **明确免责声明** - 工具仅供学习和个人使用

### 不接受的内容

以下类型的贡献不会被接受：

- 恶意代码或漏洞利用
- 未经授权访问用户数据
- 商业用途相关功能
- 违反 Bilibili 服务条款的功能
- 侵犯第三方版权的内容

## 许可证

贡献的代码将使用 MIT 许可证发布。请确保你有权提交这些代码。

## 联系方式

如有疑问，请通过以下方式联系：
- 创建 Issue
- 提交 Pull Request

再次感谢你的贡献！
